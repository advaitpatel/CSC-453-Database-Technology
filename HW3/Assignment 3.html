Part B - 6 points each

Given the schema from Figure 8.14 (or 6.14 in the 6th edition), specify the following queries in relational algebra (no SQL accepted):

  1. List all book titles.

  2. List all book titles published by "Addison Wesley".

  3. List the book titles, and branch names, at each branch where all copies of that title are available for borrowing.

  4. (b) How many copies of the book titled The Lost Tribe are owned by each library branch?

  5. (a) How many copies of the book titled The Lost Tribe are owned by the library branch whose name is "Sharpstown"?

  6. List the names of borrowers with overdue books. (Assume currentDate has today's date.)

  7. (c) Retrieve the names of all borrowers who do not have any books checked out.

  8. (d) For each book that is loaned out from the "Sharpstown" branch and whose due date is today, retrieve the book title, the borrower's name, and the borrower's address. (Assume currentDate has today's date.)

  9. (g) For each book authored (or co-authored) by "Stephen King", retrieve the title and the number of copies owned by the library branch whose name is "Central".

  10. List all book titles that have been borrowed by "Peter Bloomfield".

"List" simply implies the ouput from an operation or assigning to a result relation.
The Greek letters and other symbols are required for any credit. Zero credit for names, initials, etc.
A natural join can be indicated with either an asterisk, or a bowtie with no condition.
Use parentheses to prioritize multiple operations (do not assume a specific precedence).
You can name the result from prior problems to use in later ones if it applies, rather than rewriting the query each time.

Use the operation symbols as described in the slides:

  • Since attributes can be renamed when a result relation is created, the renames operator (as opposed to the operation) is not required.
  • All of the Greek letter symbols are available in major word processors.
  • If using a capital U for the set union symbol, use a sans serif font, like Arial, so that it appears as U, rather than U.
  • The set intersection & join symbols are available in both Word and WordPerfect.
  • For the Cartesian product you can use a capital X (� is ANSI #215, if using HTML, or under Supplemental Mathematical Operators in Word 2007).
  • As far as I can tell, the join symbol is not in available as a character in Word 2003, although it's accessible in Word 2007/2010 under Supplemental Mathematical Operators or via the Math AutoCorrect option (may require enabling) by typing "\bowtie" (as well as "\bigcup" and "\bigcap" for the set operators). Some of the other symbols may be available in Word 2007/2010 as well (and also in PowerPoint 2007/2010).
  • For what it's worth, all of the symbols, including join, union, and the logical operators, are available as characters in WordPerfect.
